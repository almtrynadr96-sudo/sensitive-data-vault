​🛡️ Secure Data Vault



​🎯 الهدف من المشروع


​تصميم وبناء بيئة قاعدة بيانات باستخدام PostgreSQL مخصصة لتخزين بيانات حساسة (مثل السجلات الطبية أو المالية)، مع تطبيق معايير أمنية صارمة تضمن سرية ونزاهة وتوافر البيانات (CIA Triad).
​🛡️ الميزات الأمنية المطبقة
​التحكم بالوصول القائم على الأدوار (RBAC): تم إنشاء أدوار مخصصة (app_user, auditor, admin) مع منح أقل صلاحيات ممكنة (Principle of Least Privilege) لكل دور، بعيداً عن استخدام حساب الـ postgres الافتراضي.
​أمن الصفوف (Row-Level Security - RLS): تطبيق سياسات تمنع المستخدمين من الوصول إلى بيانات مستخدمين آخرين على مستوى السجل، مما يضمن العزل التام للبيانات.
​التحكم بالوصول القائم على السمات (ABAC): دمج سياسات ديناميكية تعتمد على "سمات" إضافية (مثل: التوقيت، الموقع الجغرافي، أو حالة المريض) لتحديد صلاحية الوصول، مما يوفر طبقة أمان أذكى من الأدوار الثابتة.
​تشفير البيانات (Column-Level Encryption): استخدام إضافة pgcrypto لتشفير البيانات الحساسة (مثل أرقام الهوية) بحيث لا تكون مقروءة إلا عبر مفاتيح فك تشفير محددة.
​تدقيق الأمان (Audit Logging): تهيئة النظام لتسجيل كافة محاولات الدخول والعمليات الحساسة (مثل الحذف أو التعديل) لضمان القابلية للمساءلة والمراقبة المستمرة.



​🛠️ التقنيات المستخدمة
​Database: PostgreSQL 16+
​Security Extensions: pgcrypto, pgaudit
​Security Models: RBAC & ABAC implementation
​Infrastructure: Docker (لضمان بيئة تشغيل معزولة وقابلة للتكرار)

​📈 النتائج


​تم بناء نظام يحقق مبدأ "الدفاع في العمق" (Defense in Depth)، مما يقلل بشكل كبير من مخاطر تسرب البيانات الناتجة عن أخطاء التكوين أو الوصول غير المصرح به.
